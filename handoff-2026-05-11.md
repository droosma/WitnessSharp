## Key constraints to honor (from the interview)

These are not in PLAN.md as a single list — re-emphasizing them here because they shape every implementation choice:

1. **"Open for extension, closed for modification"** is the user's stated guiding principle. Sensible defaults + extension hooks; never replace idiomatic .NET surface (`IConfiguration`, `IOptions`, `ILoggerFactory`).
2. **"Lean by default, opt-in everything"** — `AddWitness` alone registers DI primitives + resource attributes only. Every instrumentation/exporter/filter is a fluent call. This is a notable shift from the reference implementation, which aggressively pre-enables a lot.
3. **`Monitor<T>` stays central** as the package's contribution. Standard primitives (`ILogger<T>`, `Meter`, `ActivitySource`) remain exposed underneath, not wrapped.
4. **Deliberate non-cleanups** — the user kept `WitnessedAction.Activity` as a public member (promoted field → property only) and kept `Finish()` despite the overlap with `Dispose()`. Don't "fix" these.
